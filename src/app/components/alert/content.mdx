## Overview

Alerts are contextual messaging components that appear inline within the interface to communicate system states, warnings, confirmations, or informational updates. Unlike modal dialogs or toast notifications, alerts remain embedded within the layout and provide persistent feedback until the issue is resolved or dismissed.

## Lifecycle / Behavior

| Phase | Behavior | Notes |
|-------|----------|-------|
| Trigger | Alert appears when a system event occurs | Form validation, API response, state change |
| Display | Alert renders inline near related content | No animation by default; fade-in optional |
| Persistence | Remains visible until resolved or dismissed | Error alerts persist until issue is fixed |
| Interaction | User reads message, clicks CTA, or dismisses | CTAs resolve the issue; dismiss hides the alert |
| Removal | Alert is removed from DOM after dismiss/resolve | Success alerts may auto-dismiss after 5–10s |

<Warning>Error and warning alerts should never auto-dismiss. They must persist until the user resolves the issue or explicitly dismisses them.</Warning>

| Alert Type | Auto-dismiss | Dismissible | Requires Action |
|------------|-------------|-------------|-----------------|
| Success | Optional (5–10s) | Yes | No |
| Error | Never | Optional | Often (CTA) |
| Warning | Never | Optional | Sometimes |
| Info | Optional (10s) | Yes | No |
| Coal/Black/White | Depends on context | Yes | No |


## Interaction Rules

| Rule | Guideline |
|------|-----------|
| Dismissible vs Persistent | Success/info = dismissible. Error/warning = persistent until resolved. |
| CTA priority | Max 2 CTAs. Primary (solid) on right, secondary (ghost) on left. |
| Stacking | Max 3 alerts visible at once. Newest on top. Same-type alerts replace, not stack. |
| Inline placement | Place directly above or below the content section the alert relates to. |
| Full-width | Alerts span the full width of their container. Never float or overlap. |
| Dismiss behavior | Trailing ✕ icon removes alert. Escape key dismisses focused alert. |
| Focus management | On dynamic insertion, move focus to error alerts. Info/success don't steal focus. |


## Types

| Type | Color | Usage |
|------|-------|-------|
| White | `#FFFFFF` | Neutral alert on standard surfaces |
| Black | `#000000` | High-contrast alert on dark environments |
| Coal | `#3D445C` | Neutral system messaging, informational |
| Success | `#1BA86E` | Successful completion of an action |
| Error | `#DC143C` | Critical issues or failures requiring attention |
| Info Blue | `#2396FB` | Informational guidance or system updates |
| Warning | `#CF9F02` | Caution or potential issues requiring awareness |


## Sizes

| Size | Target | Padding | Usage |
|------|--------|---------|-------|
| Large (Web) | Desktop, full-width sections | 16px | Comfortable spacing, longer messages |
| Small (Mobile) | Compact screens, tight layouts | 10px 12px | Readable while minimizing vertical space |


## Alert vs Toast vs Modal

| Aspect | Alert (Inline) | Toast | Modal |
|--------|---------------|-------|-------|
| Position | Inline within layout | Floating overlay (corner) | Center overlay with backdrop |
| Persistence | Persistent until resolved/dismissed | Auto-dismisses (3–5s) | Blocks until user action |
| Interaction | Optional CTAs, dismissible | Minimal (dismiss only) | Required user response |
| Use when | Contextual feedback near content | Temporary confirmations | Critical decisions, blocking actions |
| Interruption | Low — embedded in flow | Low — non-blocking | High — blocks workflow |


## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Role | Use `role="alert"` for urgent messages, `role="status"` for informational |
| Live region | Add `aria-live="assertive"` for errors, `aria-live="polite"` for info |
| Contrast | Text ≥ 4.5:1 against alert background |
| Dismiss | Close button needs `aria-label="Dismiss alert"` |
| Keyboard | Tab to focus CTAs, Escape to dismiss |
| Screen reader | Icon meaning conveyed via `aria-hidden="true"` + text label |
| Focus | Move focus to alert on dynamic insertion for errors |


## Developer Handoff

<CodeGroup>
```tsx API
interface AlertProps {
  type?: 'white' | 'black' | 'coal' | 'success' | 'error' | 'info' | 'warning';
  style?: 'single' | 'dual';
  size?: 'lg' | 'sm';
  icon?: ReactNode;
  title?: string;
  description?: string;
  badge?: ReactNode;
  dismissible?: boolean;
  onDismiss?: () => void;
  actions?: AlertAction[];
  children?: ReactNode;
}

interface AlertAction {
  label: string;
  onClick: () => void;
  variant?: 'primary' | 'secondary';
}
```

```tsx Usage
// Success alert
<Alert type="success" icon={<CheckIcon />}>
  Action completed successfully.
</Alert>

// Error with dual text
<Alert type="error" title="Payment failed" description="Your card was declined. Please update your payment method." />

// Warning with actions
<Alert
  type="warning"
  title="Unsaved changes"
  description="You have unsaved changes that will be lost."
  actions={[
    { label: 'Discard', onClick: handleDiscard, variant: 'secondary' },
    { label: 'Save Now', onClick: handleSave, variant: 'primary' }
  ]}
/>

// Dismissible info
<Alert type="info" dismissible onDismiss={() => setVisible(false)}>
  New features are available. Check the changelog.
</Alert>

// With badge
<Alert type="warning" badge={<Badge variant="warning">Review</Badge>}>
  Something needs your review.
</Alert>

// Small (mobile)
<Alert type="success" size="sm">Saved.</Alert>
```

```tsx ARIA
// Error alert with live region
<div role="alert" aria-live="assertive">
  <Alert type="error" title="Validation failed">
    Please fix the highlighted fields.
  </Alert>
</div>

// Info alert (polite)
<div role="status" aria-live="polite">
  <Alert type="info">Session expires in 5 minutes.</Alert>
</div>
```
