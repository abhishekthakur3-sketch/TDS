## Overview

Status indicators are compact visual elements that communicate real-time state information. Each indicator consists of a colored dot and a text label, making status immediately scannable across tables, lists, cards, and dashboards. The color-coded system ensures users can quickly identify success, failure, warnings, and other operational states.

| 1 | Dot | Yes | Colored circle indicating the status category |

<Warning>Both the dot and label are required. Never use a dot without a label — color alone is insufficient for accessibility.</Warning>

|------|-----------------|-------------|-----------|-----|----------|
| Large | 24px | 10px | 14px | 8px | Primary tables, detail views |
| Medium | 20px | 8px | 13px | 6px | Standard tables, lists |
| Small | 16px | 6px | 12px | 5px | Compact tables, dense layouts |
| XSmall | 12px | 5px | 11px | 4px | Inline text, metadata, tooltips |


## Status Indicator vs Badge vs Tag

| Feature | Status Indicator | Badge | Tag |
|---------|-----------------|-------|-----|
| Purpose | Real-time state | Count or label | Categorization |
| Interactive | No | No | Sometimes (removable) |
| Visual | Dot + text | Filled pill | Filled/outlined chip |
| Best for | Process status, system state | Notification counts, labels | Filtering, categorization |
| Updates | Real-time | On event | Static or user-driven |


## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Color independence | Always include a text label — never rely on color alone |
| Screen readers | Use `aria-label` or visible text to announce the status |
| Contrast | Dot colors must meet 3:1 contrast ratio against background |
| Role | Use `role="status"` for live-updating indicators |
| Live regions | Add `aria-live="polite"` for real-time status changes |
| Decorative dot | The dot is decorative — hide from screen readers with `aria-hidden="true"` |

<Info>The text label is the primary accessible element. The colored dot is supplementary visual information and should be hidden from screen readers.</Info>


## Developer Handoff

<CodeGroup>
```tsx API
import { StatusIndicator } from '@tarmac/design-system';

interface StatusIndicatorProps {
  variant: 'success' | 'failed' | 'warning' | 'information'
    | 'synced' | 'scheduled' | 'unknown' | 'pause'
    | 'play' | 'downloading' | 'pending';
  size?: 'large' | 'medium' | 'small' | 'xsmall'; // Default: 'medium'
  label?: string;                                    // Custom label override
  live?: boolean;                                    // Enable aria-live for real-time updates
}
```

```tsx Usage
// Basic usage
<StatusIndicator variant="success" />
<StatusIndicator variant="failed" />
<StatusIndicator variant="pending" size="small" />

// Custom label
<StatusIndicator variant="success" label="Delivered" />
<StatusIndicator variant="downloading" label="Syncing data..." />

// In a table
<table>
  <tbody>
    <tr>
      <td>#ORD-4821</td>
      <td><StatusIndicator variant="success" size="small" /></td>
      <td>2 min ago</td>
    </tr>
    <tr>
      <td>#ORD-4822</td>
      <td><StatusIndicator variant="failed" size="small" /></td>
      <td>5 min ago</td>
    </tr>
  </tbody>
</table>

// Live-updating status
<StatusIndicator variant="downloading" live />
```
