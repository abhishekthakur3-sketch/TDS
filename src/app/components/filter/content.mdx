## Overview

The Filter system provides structured data filtering across web and mobile platforms. It includes Filter Inputs (trigger buttons), Filter List Cells (selectable options), Filter Lists (grouped options), Filter Dropdowns (collapsed/expanded selectors), and a Filter Set Bottom Sheet for mobile. Together they enable users to refine data views efficiently.

## Filter Input States

| State | Description |
|-------|-------------|
| Default | Neutral, ready for interaction |
| Hover | Pointer over the input |
| Active | Filter has active selections |
| Focused | Keyboard focus active |
| Disabled | Filter unavailable |
| Ghost | Skeleton/loading state |


## Filter Dropdown Variants

| Variant | Description |
|---------|-------------|
| Collapsed | Input visible, list hidden |
| Expanded | Input visible, list open below |
| Active | Filter has active selections applied |


## Sizes

| Size | Input Height | Cell Height | Usage |
|------|-------------|-------------|-------|
| Large | 44px | 44px | Standard filter bars, prominent filters |
| Small | 40px | 40px | Compact layouts, dense filter rows |


## Usage Guidelines

<CardGroup cols={2}>
  <Card title="Do" icon="check" color="#16A34A">
    - Use filters for narrowing large data sets
    - Keep filter labels short and descriptive
    - Show active filter count or selection
    - Use bottom sheet for mobile filter experiences
    - Allow clearing all filters easily
  </Card>
  <Card title="Don't" icon="xmark" color="#DC2626">
    - Don't overload with too many filter options
    - Don't hide critical filters behind secondary menus
    - Don't use filters for navigation purposes
    - Don't mix filter and sort controls without clear separation
    - Don't use filters when the data set is small
  </Card>
</CardGroup>
