## Overview

The Search component is an input designed specifically for searching. It enables users to quickly locate information by entering keywords or phrases. Search fields often trigger dynamic suggestions, filters, or result lists via a dropdown. The system is built from three foundational elements: Search Field, Search Dropdown Cells, and Search Dropdown Lists.

## Sub-Component: Search Dropdown Cells

Individual search result items displayed inside the dropdown. Each cell may contain icons, text, or subtext.

| Property | Options | Description |
|----------|---------|-------------|
| Style | Regular, Blue | Visual differentiation for highlighted results |
| State | Default, Hover, Pressed, Disabled | Selection feedback |
| Size | Default, Large, Small | Matches search field context |
| Leading Icon | Boolean | Icon before result text |
| Trailing Icon | Boolean | Action icon after result text |
| Subtext | Boolean | Secondary description line |


## Search Dropdown

The combined component pairing the search field with a results dropdown.

| Property | Options | Description |
|----------|---------|-------------|
| Variant | Collapsed, Expanded | Dropdown visibility state |
| Size | Default, Large, Small | Dimensional scale |

## Sizes

| Size | Field Height | Cell Height | Usage |
|------|-------------|-------------|-------|
| Large | 48px | 70px | Prominent search bars, full-page search |
| Default | 44px | 62px | Standard navigation search, filters |
| Small | 36px | 54px | Compact layouts, inline search |


## Behavior

| Interaction | Result |
|-------------|--------|
| Click search field | Places cursor, ready for input |
| Type query | Triggers suggestions/results dynamically |
| Enter key | Confirms search query |
| Clear input | Resets search results |
| Arrow keys (in dropdown) | Navigate between results |
| Click result | Selects and applies the result |
| Click outside | Closes dropdown |
| Escape | Closes dropdown |


## Usage Guidelines

<CardGroup cols={2}>
  <Card title="✓ Do" icon="check" color="#16A34A">
    - Provide clear placeholder text (e.g., "Search...")
    - Use a search icon to indicate purpose
    - Support keyboard input and navigation
    - Allow users to clear the search easily
    - Group dropdown results logically
  </Card>
  <Card title="✗ Don't" icon="xmark" color="#DC2626">
    - Don't replace labels with placeholder text only
    - Don't hide search results without feedback
    - Don't overload search fields with too many controls
    - Don't display too many results at once
    - Don't use for non-search input purposes
  </Card>
</CardGroup>
