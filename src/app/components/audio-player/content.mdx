## Overview

Audio Player allows users to record, preview, and manage audio inputs. It transitions smoothly between states — from idle recording to active playback — while displaying progress, duration, and action controls.

## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Labels | All control buttons must have `aria-label` (e.g. "Start recording", "Play audio") |
| Keyboard | Full keyboard support — Tab to focus, Enter/Space to activate |
| Screen readers | Announce state changes (recording started, paused, saved) |
| Progress | Use `aria-valuenow`, `aria-valuemin`, `aria-valuemax` on the progress bar |
| Non-visual feedback | Timer and status labels must be readable by assistive tech |
| Visible states | Ensure each variant has a visually distinct appearance |


## Developer Handoff

<CodeGroup>
```tsx Component API
interface AudioPlayerProps {
  variant?: 'default' | 'recording' | 'play' | 'pause' | 'saved' | 'disabled';
  size?: 'large' | 'small';
  style?: 'light' | 'dark';
  helperTextTop?: string;
  helperTextBottom?: string;
  duration?: string;        // e.g. "00:48"
  progress?: number;        // 0–100
  onRecord?: () => void;
  onPlay?: () => void;
  onPause?: () => void;
  onSave?: () => void;
  onDelete?: () => void;
}
```

```tsx Usage
import { AudioPlayer } from '@tarmac/design-system';

// Default — ready to record
<AudioPlayer
  variant="default"
  size="large"
  style="light"
  helperTextTop="Record your instruction"
  helperTextBottom="Max 60 seconds"
  onRecord={() => startRecording()}
/>

// Playback state
<AudioPlayer
  variant="play"
  size="large"
  style="light"
  duration="00:48"
  progress={50}
  onPlay={() => playAudio()}
  onDelete={() => deleteRecording()}
/>

// Saved state
<AudioPlayer
  variant="saved"
  size="small"
  style="dark"
  duration="00:48"
  onPlay={() => playAudio()}
  onDelete={() => deleteRecording()}
/>

// Disabled
<AudioPlayer
  variant="disabled"
  size="large"
  style="light"
/>
```
