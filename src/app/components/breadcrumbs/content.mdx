## Overview

Breadcrumbs display a navigational trail that helps users understand their current location within a product hierarchy. Each breadcrumb represents a navigable level, with the final item representing the current page. The system consists of two components: Breadcrumb Cells (individual items) and Breadcrumb Navigation (the assembled path).

## Anatomy

|---|---------|----------|-------------|
| 1 | Leading Icon | No | Icon before the label for visual context |
| 2 | Title | Yes | Primary label text — the only required element |
| 3 | Trailing Icon | No | Icon after the label (navigation direction, dropdown) |
| 4 | Pills | No | Contextual metadata or status indicator |

### Breadcrumb Navigation

| # | Element | Required | Description |
|---|---------|----------|-------------|
| 1 | Breadcrumbs Container | Yes | Wrapper with horizontal layout |
| 2 | First Breadcrumb Cell | Yes | Root-level navigation item |
| 3 | Separator / Divider | Yes | Slash or Chevron between items |
| 4 | Divider Spacing | Yes | Consistent gap around separators |
| 5 | Last Breadcrumb Cell | Yes | Current page (non-interactive) |

<Warning>Only the Title is required in each breadcrumb cell. Leading icons, trailing icons, and pills are optional and should be used sparingly.</Warning>


## Styles

| Style | Link Color | Hover Background | Use Case |
|-------|-----------|-----------------|----------|
| Black (Default) | `#000000` | `#F5F5F5` | Standard navigation, neutral contexts |
| Blue | `#2396FB` | `#EBF5FF` | Interactive/informational contexts |
| DLV Red | `#ED1B36` | `#FDEAED` | Brand-specific or high-emphasis contexts |


## States

| State | Visual Treatment | Interaction |
|-------|-----------------|-------------|
| Default | Normal text, link styling | Clickable, navigates to page |
| Hover | Underline + subtle background tint | Pointer cursor |
| Pressed | Darker background tint | Active click |
| Disabled | 30% opacity | Not interactive, `cursor: not-allowed` |
| Ghost | Reduced opacity, no underline | Current page — non-interactive |


## Behavior

### Navigation

- All breadcrumbs except the last are clickable navigation links
- The final breadcrumb represents the current page and is non-interactive (Ghost state)
- Clicking a breadcrumb navigates to that hierarchy level
- Breadcrumbs must remain consistent with the product's navigation structure

### Truncation

- Long breadcrumb labels should be truncated with ellipsis (`...`)
- On narrow screens, middle breadcrumbs can collapse to `...` with a dropdown to reveal hidden items
- The first (root) and last (current) breadcrumbs should always remain visible

### Optional Elements

| Element | Behavior |
|---------|----------|
| Leading Icon | Displays before the label for visual context (e.g., home icon on root) |
| Trailing Icon | Appears after the label, may indicate dropdown or navigation direction |
| Pills | Shows contextual metadata or status (e.g., count, status label) |


## Usage Guidelines

<CardGroup cols={2}>
  <Card title="Do" icon="check" color="#16A34A">
    - Use breadcrumbs for hierarchical navigation structures
    - Keep labels short and descriptive (1–3 words)
    - Always show the current page as the last non-interactive item
    - Use consistent divider style across the product
    - Place breadcrumbs at the top of the page, below the header
    - Start with a root item (e.g., "Home" or "Dashboard")
  </Card>
  <Card title="Don't" icon="xmark" color="#DC2626">
    - Don't use breadcrumbs for flat navigation structures
    - Don't make the current page breadcrumb clickable
    - Don't use breadcrumbs as the primary navigation system
    - Don't mix slash and chevron dividers in the same product
    - Don't exceed 8 breadcrumb levels
    - Don't use icons as the sole label — always include text
  </Card>
</CardGroup>
