## Overview

Accordions are interactive disclosure components used to organize content into expandable sections. They reduce visual clutter by displaying only section headers initially, allowing users to progressively reveal content when needed. Accordions are commonly used in settings panels, FAQs, content summaries, and form sections.

| # | Element | Required | Description |
|---|---------|----------|-------------|
| 1 | Container | Yes | Header surface with padding and border |
| 2 | Leading Icon | No | Icon before text for visual context |
| 3 | Primary Text | Yes | Section title |
| 4 | Badge | No | Status label or metadata |
| 5 | Status Indicator | No | Visual dot for state/category |
| 6 | Status Text | No | Supporting count or status text |
| 7 | Trailing Icon | Yes | Chevron indicating expand/collapse state |


## Sizes

| Size | Header Height | Padding | Usage |
|------|--------------|---------|-------|
| Large | 48px | 14px 16px | Desktop, spacious layouts |
| Small | 44px | 10px 12px | Mobile, compact layouts |


## States

| State | Visual | Behavior |
|-------|--------|----------|
| Collapsed | Chevron pointing down, content hidden | Default state |
| Expanded | Chevron pointing up, content visible | Section is open |
| Hover | Subtle background tint on header | Indicates interactivity |
| Focused | 2px focus ring on header | Keyboard navigation |
| Disabled | 40% opacity, non-interactive | Section unavailable |


## Section Count

| Count | Use Case |
|-------|----------|
| 1 | Single collapsible section (rare) |
| 2 | Binary content grouping |
| 3 | Common — settings, FAQ, checkout |
| 4 | Detailed configuration panels |
| 5 | Maximum recommended per group |

<Warning>Avoid more than 5 accordion sections in a single group. If content requires more, consider tabs or separate pages.</Warning>


## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Keyboard | Enter/Space to toggle; Arrow keys to navigate between headers |
| ARIA | `aria-expanded="true/false"` on header button |
| Role | Header uses `<button>`, content uses `role="region"` |
| Labelling | Content region linked via `aria-labelledby` to header |
| Focus | Visible focus ring on header during keyboard navigation |
| Screen reader | Announces expanded/collapsed state on toggle |


## Developer Handoff

<CodeGroup>
```tsx API
interface AccordionProps {
  size?: 'lg' | 'sm';
  allowMultiple?: boolean;
  defaultExpanded?: string[];
  children: ReactNode;
}

interface AccordionItemProps {
  id: string;
  title: string;
  subtitle?: string;
  icon?: ReactNode;
  badge?: ReactNode;
  statusText?: string;
  disabled?: boolean;
  children: ReactNode;
}
```

```tsx Usage
// Basic accordion
<Accordion>
  <AccordionItem id="shipping" title="Shipping information">
    Standard delivery takes 3–5 business days.
  </AccordionItem>
  <AccordionItem id="payment" title="Payment details">
    We accept credit cards, debit cards, and UPI.
  </AccordionItem>
  <AccordionItem id="summary" title="Order summary">
    3 items in your cart totaling ₹2,450.
  </AccordionItem>
</Accordion>

// With optional elements
<Accordion size="lg" allowMultiple>
  <AccordionItem id="ship" title="Shipping" icon={<BoxIcon />} badge={<Badge variant="success">Active</Badge>}>
    Content here...
  </AccordionItem>
  <AccordionItem id="pay" title="Payment" icon={<CardIcon />} badge={<Badge variant="warning">Pending</Badge>}>
    Content here...
  </AccordionItem>
</Accordion>

// Single expand mode
<Accordion allowMultiple={false} defaultExpanded={['faq-1']}>
  <AccordionItem id="faq-1" title="What is the return policy?">
    You can return items within 30 days.
  </AccordionItem>
  <AccordionItem id="faq-2" title="How do I track my order?">
    Use the tracking link sent to your email.
  </AccordionItem>
</Accordion>
```

  </div>
</div>
