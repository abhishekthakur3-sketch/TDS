## Overview

Links are lightweight navigational elements embedded within text, lists, or UI structures. Unlike buttons, links represent navigation rather than actions. They support optional leading and trailing icons for additional visual context such as external navigation or directional cues. Links should remain visually subtle while clearly indicating interactivity.

| # | Element | Required | Description |
|---|---------|----------|-------------|
| 1 | Leading Icon | No | Icon before the label for visual context |
| 2 | Label | Yes | Clickable text describing the navigation destination |
| 3 | Trailing Icon | No | Icon after the label (external link indicator, arrow, etc.) |

<Warning>Icons should support the link meaning but never replace the text label. The label is always required.</Warning>

| XLarge | 32px | 20px | 24px | Hero sections, prominent CTAs |
| Large | 26px | 17px | 20px | Section headers, featured content |
| Medium | 24px | 15px | 18px | Body text, standard navigation |
| Small | 20px | 13px | 16px | Supporting text, metadata |
| XSmall | 16px | 11px | 14px | Footnotes, dense layouts, captions |


## Behavior

### Navigation

- Clicking a link navigates to the target location
- Links may open content in the same page, a new tab, or an external destination
- Use `target="_blank"` with `rel="noopener noreferrer"` for external links
- Disabled links remain visible but cannot be interacted with

### Icon Usage

| Pattern | Example | Use Case |
|---------|---------|----------|
| Leading icon | 📄 View document | Contextual icon before label |
| Trailing icon | Open in new tab ↗ | External link indicator |
| Both icons | 📄 View document ↗ | Combined context + direction |
| No icons | Learn more | Simple inline navigation |

### Inline vs Standalone

- Inline links are embedded within body text and should match the surrounding font size
- Standalone links appear independently and can use any size
- Inline links should use underline on default state for discoverability within text


## Usage Guidelines

<CardGroup cols={2}>
  <Card title="Do" icon="check" color="#16A34A">
    - Use descriptive link labels that explain the destination
    - Maintain visual distinction between links and regular text
    - Ensure adequate color contrast against the background
    - Use trailing icons to indicate external navigation
    - Keep link text concise and actionable
    - Match link size to surrounding typography
  </Card>
  <Card title="Don't" icon="xmark" color="#DC2626">
    - Don't use links for primary actions (use buttons)
    - Don't use vague labels like "Click here" or "Read more"
    - Don't hide link meaning behind icons only
    - Don't use links without clear context
    - Don't use Blue links on blue backgrounds
    - Don't remove underline from inline links in body text
  </Card>
</CardGroup>
