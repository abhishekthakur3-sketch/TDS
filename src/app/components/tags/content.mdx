## Overview

Tags are compact UI elements used to represent attributes, filters, or categories associated with content. They help users quickly identify grouped information and interact with filtering or classification systems. Unlike badges, tags are often interactive and may support actions such as removal or filtering. Tags use a rectangular shape with `border-radius: 4px`, distinguishing them from the fully rounded pill component.

| Pattern | Behavior | Use Case |
|---------|----------|----------|
| Selectable (single) | One tag active at a time, others deselect | Sort order, view mode |
| Selectable (multi) | Multiple tags can be active simultaneously | Category filters, skill tags |
| Removable | Tag dismissed via close icon (✕) | Active filters, applied selections |
| Toggle | Tag switches between on/off state | Feature toggles, preferences |
| Read-only | Non-interactive, informational only | Metadata labels, status indicators |

<Warning>For selectable tag groups, clearly indicate the selected state using filled style for active and outlined for inactive.</Warning>


## Content Rules

| Rule | Guideline |
|------|-----------|
| Max text length | 1–3 words recommended, 20 characters max |
| Truncation | Ellipsis (…) after max-width, show full text in tooltip |
| Overflow | Tags wrap to next line in groups, never horizontally scroll |
| Casing | Sentence case for labels ("In progress" not "IN PROGRESS") |
| Icons | Leading icon max 16px, trailing close icon 12px |
| Empty state | Hide tag group entirely if no tags to display |


## Color Types

"Type" refers to the color variant of the tag, communicating semantic meaning.

| Type | Color | Usage |
|------|-------|-------|
| Black | `#000000` | Strong emphasis, neutral system tags |
| White | `#FFFFFF` | Dark backgrounds, inverted themes |
| Coal | `#3D445C` | System metadata, generic classifications |
| Blue | `#2396FB` | Informational, contextual guidance |
| Success | `#1BA86E` | Positive states, completed items |
| Error | `#DC143C` | Critical issues, failed items |
| Warning | `#CF9F02` | Caution, attention-required items |
| Legacy Blue | `#0066CC` | Legacy system compatibility |
| DLV Red | `#ED1B36` | Delhivery brand-specific states |


## Sizes

| Size | Height | Font Size | Padding | Usage |
|------|--------|-----------|---------|-------|
| XLarge | 36px | 14px | 0 12px | Hero sections, prominent filters |
| Large | 28px | 12px | 0 10px | Cards, banners, filter bars |
| Medium | 24px | 11px | 0 8px | Default — lists, tables, headers |
| Small | 20px | 10px | 0 6px | Dense layouts, metadata rows |

## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Contrast | Text ≥ 4.5:1 against background |
| Color independence | Don't rely on color alone — include text labels |
| Screen reader | Use `aria-label` for icon-only or dot-only tags |
| Removable tags | Close button needs `aria-label="Remove [tag name]"` |
| Keyboard | Tab to focus, Enter/Space to activate, Delete to remove |
| Focus ring | Visible 2px focus ring on keyboard navigation |
| Role | Use `role="option"` in filter groups, `role="listbox"` for container |
| Selected state | Use `aria-selected="true"` for active tags in groups |


## Developer Handoff

<CodeGroup>
```tsx API
interface TagProps {
  type?: 'black' | 'white' | 'coal' | 'blue' | 'success' | 'error' | 'warning' | 'legacy_blue' | 'dlv_red';
  variant?: 'standard' | 'outlined';
  size?: 'xl' | 'lg' | 'md' | 'sm';
  state?: 'default' | 'hover' | 'pressed' | 'focused' | 'disabled' | 'ghost';
  selected?: boolean;
  icon?: ReactNode;
  closable?: boolean;
  onClose?: () => void;
  onClick?: () => void;
  dot?: boolean;
  children?: ReactNode;
}
```

```tsx Usage
// Standard tags
<Tag type="black">Category</Tag>
<Tag type="success">Active</Tag>
<Tag type="error">Failed</Tag>

// Outlined style
<Tag type="blue" variant="outlined">Draft</Tag>

// Sizes
<Tag type="black" size="xl">XLarge</Tag>
<Tag type="black" size="sm">Small</Tag>

// With leading icon
<Tag type="success" icon={<CheckIcon />}>Verified</Tag>

// Removable tag with close icon
<Tag type="coal" closable onClose={() => handleRemove()}>React</Tag>

// With status dot
<Tag type="success" dot>Online</Tag>

// Selected state
<Tag type="blue" selected>Active Filter</Tag>

// Disabled
<Tag type="black" state="disabled">Unavailable</Tag>
```

```tsx Filter Group
// Removable filter tags
<TagGroup aria-label="Active filters">
  <Tag type="blue" closable>Status: Active</Tag>
  <Tag type="warning" closable>Priority: High</Tag>
  <Tag type="success" closable>Region: North</Tag>
</TagGroup>

// Single-select group
<TagGroup selectable mode="single" aria-label="Sort order">
  <Tag type="black" selected>Newest</Tag>
  <Tag type="black" variant="outlined">Oldest</Tag>
  <Tag type="black" variant="outlined">Popular</Tag>
</TagGroup>

// Multi-select group
<TagGroup selectable mode="multi" aria-label="Categories">
  <Tag type="blue" selected>React</Tag>
  <Tag type="blue" selected>TypeScript</Tag>
  <Tag type="blue" variant="outlined">Vue</Tag>
  <Tag type="blue" variant="outlined">Angular</Tag>
</TagGroup>
```
