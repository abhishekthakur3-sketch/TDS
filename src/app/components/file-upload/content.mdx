## Overview

The File Upload component enables users to upload files through drag-and-drop or manual selection. It provides structured feedback across the upload lifecycle including idle, uploading, success, and error states. The system combines an Upload Container (UI surface) with Upload File items (individual file representations) and Upload Illustrations for visual feedback.

## Upload Container

The primary interactive area that accepts file input.

| Variant | Description |
|---------|-------------|
| Default (Idle) | Empty state with drag-and-drop zone and browse action |
| Uploading | One or more files in progress |
| Added (Completed) | All files successfully uploaded with management actions |
| Uploading + Added | Multi-file: some completed, some still uploading |

## Behavior

| Interaction | Result |
|-------------|--------|
| Drag file over container | Drop zone highlights |
| Drop file | Upload begins, file item appears |
| Click "Browse" | System file dialog opens |
| Upload in progress | Progress bar animates, cancel available |
| Upload complete | File item shows completed state with actions |
| Upload fails | Error state with retry option |
| Click remove | File item removed from list |
| Click change | Replace file via system dialog |


## Accessibility

| Requirement | Guideline |
|-------------|-----------|
| Keyboard | Browse button must be keyboard accessible |
| Focus | Visible focus ring on interactive elements |
| Screen reader | Announce upload state changes (uploading, complete, error) |
| Labels | Provide descriptive labels for all action buttons |
| Progress | Announce progress updates via `aria-live` |


## Developer Handoff

<CodeGroup>
```tsx API
interface FileUploadProps {
  variant?: 'default' | 'uploading' | 'added' | 'uploading_added';
  files?: UploadFile[];
  onFilesChange?: (files: UploadFile[]) => void;
  onUpload?: (file: File) => void;
  onRemove?: (fileId: string) => void;
  onCancel?: (fileId: string) => void;
  accept?: string;
  maxSize?: number;
  multiple?: boolean;
  title?: string;
  subtext?: string;
}

interface UploadFile {
  id: string;
  name: string;
  size: number;
  status: 'pending' | 'uploading' | 'completed' | 'failed' | 'warning';
  progress?: number;
  error?: string;
}
```

```tsx Usage
// Default empty state
<FileUpload
  title="Upload your files here"
  subtext="Supports PDF, PNG, JPG up to 15MB"
  onUpload={handleUpload}
/>

// With uploaded files
<FileUpload
  variant="added"
  files={[
    { id: '1', name: 'document.pdf', size: 15800000, status: 'completed' },
    { id: '2', name: 'image.png', size: 2400000, status: 'completed' },
  ]}
  onRemove={handleRemove}
/>

// Multi-file with mixed states
<FileUpload
  variant="uploading_added"
  multiple
  files={[
    { id: '1', name: 'report.pdf', size: 8000000, status: 'completed' },
    { id: '2', name: 'data.csv', size: 12000000, status: 'uploading', progress: 45 },
  ]}
/>
```
