## Overview

A grid system is the invisible structural framework that organizes layout, spacing, and alignment across your product. It ensures consistency, predictability, and rhythm — making interfaces feel balanced, intuitive, and easy to scan. A strong grid acts like a "traffic system" for UI elements: guiding where components should sit, how they scale, and how they respond across breakpoints.

## Grid Usage Guidelines

Choosing the right grid configuration depends on the content type, density, and user task. Use this decision framework to select the appropriate layout.

<Tabs>
  <Tab title="When to Use Fluid vs Fixed">
    | Scenario | Grid Type | Reason |
    |----------|-----------|--------|
    | Dashboards & analytics | Fluid | Maximize data visibility across screen widths |
    | Forms & data entry | Fixed (narrow) | Constrain width for comfortable reading and input |
    | Marketing / landing pages | Fixed (wide) | Control line length and visual composition |
    | Data tables & lists | Fluid | Tables need horizontal space to avoid truncation |
    | Settings / configuration | Fixed (narrow) | Focused, single-task workflows benefit from constraints |
    | Multi-panel workflows | Fluid | Side panels + content need proportional scaling |
  </Tab>
  <Tab title="Column Strategy by Context">
    | Context | Desktop Columns | Mobile Columns | Notes |
    |---------|----------------|----------------|-------|
    | Full-width dashboard | 12 | 4 | Cards span 3-4 cols each on desktop |
    | Form with sidebar | 8 (form) + 4 (sidebar) | 4 (stacked) | Sidebar collapses below form on mobile |
    | Data table | 12 | 4 (horizontal scroll) | Tables maintain structure, scroll horizontally |
    | Article / documentation | 8 (content) + 4 (TOC) | 4 (TOC hidden) | Table of contents hides on mobile |
    | Modal / dialog | 6-8 (centered) | 4 (full width) | Modals use a subset of the grid |
    | Login / auth screens | 4-6 (centered) | 4 | Narrow, focused layout |
  </Tab>
</Tabs>


## Types of Grid

<Tabs>
  <Tab title="Fluid Grid">
    A fluid grid stretches across the screen to fill all available space. Columns and content scale and resize proportionally with the viewport.

    **When to use:**
    - Responsive layouts that need to adapt to any screen width
    - Content-heavy pages where maximizing reading area matters
    - Dashboards and data-dense interfaces

    <Info>Fluid grids are the default for most TARMAC layouts. They ensure content fills the available space while maintaining column proportions.</Info>
  </Tab>
  <Tab title="Fixed Grid">
    A fixed grid applies an ideal maximum width to page containers. Content is centered within the viewport, and the grid does not stretch beyond its defined max-width.

    **Variants:**
    - **Fixed Narrow** — For focused reading experiences and forms
    - **Fixed Wide** — For content-rich layouts with sidebars

    **When to use:**
    - Marketing and landing pages where line length control matters
    - Form-heavy workflows that benefit from constrained width
    - Screens where content readability is more important than space utilization

    <Info>On viewports wider than 1440px, the TARMAC grid locks to a 1280px centered container to prevent eye-strain from overly long line lengths.</Info>
  </Tab>
</Tabs>

## General Principles

<CardGroup cols={2}>
  <Card title="Consistency over complexity" icon="equals">
    Apply the same visual logic across all surfaces. A predictable grid reduces cognitive load for both users and builders.
  </Card>
  <Card title="Content-first alignment" icon="align-left">
    Grids support meaning — they don't compete with it. Layout decisions should serve the content hierarchy, not override it.
  </Card>
  <Card title="Predictable navigation zones" icon="compass">
    Users should always know where to look. Consistent placement of navigation, content, and actions builds spatial memory.
  </Card>
  <Card title="Scalable spacing system" icon="ruler-combined">
    Built from TARMAC spacing tokens. Grid gutters, margins, and offsets reference the same scale used across all components.
  </Card>
  <Card title="Clarity at operational scale" icon="binoculars">
    Interfaces must remain readable in real-world environments — from compact POS terminals to wide enterprise dashboards.
  </Card>
  <Card title="Grid stability" icon="lock">
    Layouts should not jump, shift, or realign unexpectedly. Transitions between breakpoints and navigation states must feel smooth and intentional.
  </Card>
</CardGroup>

<Warning>Never break the grid to "make something fit." If content doesn't align to the grid, revisit the content structure or component sizing — not the grid itself.</Warning>
