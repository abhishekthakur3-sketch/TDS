## Responsive Behavior Rules

How layouts adapt across breakpoints. These rules govern what scales, what stays fixed, and how content reflows.


  <Accordion title="Column Collapse Behavior" icon="arrows-down-to-line">
    As viewport width decreases, columns collapse and content stacks vertically:
    - **Desktop → Tablet**: 12 columns maintained, column width reduces, gutters shrink from 24px to 16px
    - **Tablet → Mobile**: 12 columns collapse to 4 columns, multi-column layouts stack vertically
    - **Side panels**: Collapse into full-width sections or slide-in drawers on mobile
    - **Navigation**: Side nav becomes a hamburger menu overlay on mobile
  </Accordion>
  <Accordion title="What Scales vs What Stays Fixed" icon="ruler-combined">
    | Element | Behavior | Notes |
    |---------|----------|-------|
    | Column width | Scales (fluid) | Proportional to viewport |
    | Gutter width | Fixed per breakpoint | 24px (large) or 16px (default/small) |
    | Margins | Fixed per breakpoint | 16px on mobile, 80px+ on desktop |
    | Content max-width | Fixed at 1280px | Prevents overly wide layouts on 1920+ |
    | Font sizes | Fixed per breakpoint | Controlled by typography tokens, not grid |
    | Touch targets | Fixed minimum 44px | Never scale below minimum |
  </Accordion>
  <Accordion title="Content Stacking Logic" icon="layer-group">
    When columns collapse on smaller viewports, content stacks in reading order (top to bottom):

    1. **Primary content** stacks first (main body, forms, data)
    2. **Secondary content** stacks below (sidebars, filters, metadata)
    3. **Navigation** moves to overlay or bottom bar
    4. **Actions** (buttons, CTAs) remain visible — never hidden by stacking

    <Warning>Never hide critical actions or primary content when stacking. If content doesn't fit, simplify the layout — don't just stack everything.</Warning>
  </Accordion>



## Mobile Grid System

| Screen Size | Usage |
|-------------|-------|
| 390 × 844 | Default reference size |
| 360 × 800 | Secondary modern devices |
| 320 × 568 | POS and constrained environments |

### Main Content Grid

The mobile content grid uses 4 columns across all breakpoints, providing a simple and consistent layout framework.

| Breakpoint | Viewport | Columns | Gutter | Margin |
|------------|----------|---------|--------|--------|
| Large | 390 × 844 | 4 | 16px | 16px |
| Default | 360 × 800 | 4 | 16px | 16px |
| Small | 320 × 568 | 4 | 16px | 16px |

<Info>Mobile gutters and margins remain fixed at 16px across all breakpoints. The columns themselves are fluid and resize to fill the available width.</Info>

## Real Usage Examples


  <Accordion title="Dashboard Layout" icon="chart-line">
    - **Desktop**: 12-column grid. KPI cards span 3 cols each (4 per row). Charts span 6 cols (2 per row). Activity feed spans 4 cols as a sidebar.
    - **Tablet**: KPI cards span 6 cols (2 per row). Charts stack to 12 cols. Sidebar collapses to full-width section below.
    - **Mobile**: All cards span 4 cols (full width), stacked vertically. Charts use full width with horizontal scroll for dense data.
    - **Gutters**: 16px between cards. 24px between major sections.
  </Accordion>
  <Accordion title="Form Layout" icon="rectangle-list">
    - **Desktop**: Form spans 8 cols (centered) or 8 cols with 4-col help sidebar. Input fields span 4 cols each (2 per row) or 8 cols (full width for text areas).
    - **Mobile**: All fields span 4 cols (full width). Labels stack above inputs. Submit button spans full width.
    - **Spacing**: 16px between form fields. 24px between form sections. 32px before submit actions.
  </Accordion>
  <Accordion title="Data Table Layout" icon="table">
    - **Desktop**: Table spans 12 cols. Columns align to grid columns where possible. Filters in a 3-col sidebar or full-width bar above.
    - **Mobile**: Table spans 4 cols with horizontal scroll. Key columns (name, status) visible; secondary columns accessible via scroll. Filters collapse into a modal or accordion.
    - **Alignment**: Table headers align to grid. Cell padding uses spacing tokens (12px horizontal, 8px vertical).
  </Accordion>
  <Accordion title="Navigation Layout" icon="bars">
    - **Desktop (Side Nav Expanded)**: Nav occupies ~280px (4-col equivalent). Content grid shifts right with 12 columns in remaining space.
    - **Desktop (Side Nav Collapsed)**: Nav rail at ~72-88px. Content grid expands.
    - **Mobile**: Side nav becomes a slide-in overlay using 3-column grid. Top bar uses 2-column grid (logo left, actions right). Bottom bar uses 2-column grid for tab navigation.
  </Accordion>
